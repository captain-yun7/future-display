---

## 🎤 인사 및 프로젝트 소개 (30초)

안녕하세요. 오늘 소개할 프로젝트는 **"차세대 디스플레이 학습 웹사이트"**입니다.

이 프로젝트는 충북보건과학대학교 COSS혁신융합대학사업단에서 디스플레이 기술 교육을 목적으로 개발한 웹 애플리케이션입니다.

학생들이 CRT부터 MicroLED까지, 다양한 디스플레이 기술을 쉽고 재미있게 학습할 수 있도록 인터랙티브한 웹 플랫폼을 구축했습니다.

## 🛠 기술 스택 및 아키텍처 (1분)

### 기술 스택 선정 이유

먼저 기술 스택에 대해 말씀드리겠습니다.

**프론트엔드**는 **Next.js 15**와 **React 19**를 사용했습니다.
- Next.js의 App Router를 활용하여 각 디스플레이 기술별로 독립적인 페이지를 동적 라우팅으로 구현했습니다.
- SSR과 SSG를 통해 SEO를 최적화하고, 초기 로딩 속도를 개선했습니다.

**TypeScript**를 도입하여 타입 안전성을 확보했습니다.
- 디스플레이 기술 데이터의 구조를 인터페이스로 정의하여, 런타임 에러를 사전에 방지했습니다.

**스타일링**은 **Tailwind CSS**를 사용했습니다.
- 유틸리티 퍼스트 방식으로 빠른 개발이 가능했고, 반응형 디자인 구현이 용이했습니다.

**애니메이션**은 **Framer Motion**을 사용했습니다.
- 부드러운 페이지 전환, 스크롤 애니메이션, 호버 효과 등을 선언적으로 구현할 수 있었습니다.

### 프로젝트 구조

프로젝트는 Next.js의 App Router 구조를 따릅니다.
- `app/` 디렉토리에는 페이지 컴포넌트들이 있고
- `components/`는 layout, displays, ui, common으로 나눠 재사용 가능한 컴포넌트들을 관리합니다.
- `lib/`에는 디스플레이 데이터와 전역 상수, 유틸리티 함수들이 있습니다.

## 📚 핵심 기능 및 구현 (2분)

### 1. 디스플레이 기술 카탈로그

이 웹사이트는 총 **6가지 디스플레이 기술**을 다룹니다:
1. **CRT** - 과거의 브라운관 기술
2. **LCD** - 현재 가장 널리 사용되는 액정 디스플레이
3. **LED** - 대형 디스플레이에 주로 사용
4. **OLED** - 프리미엄 스마트폰과 TV에 사용되는 자체발광 디스플레이
5. **QLED** - 삼성의 퀀텀닷 기술
6. **MicroLED** - 차세대 디스플레이 기술

각 기술에 대해 다음과 같은 정보를 제공합니다:

### 2. 데이터 구조화

```typescript
export interface DisplayTech {
  id: string;              // 고유 식별자
  name: string;            // 약칭 (예: OLED)
  fullName: string;        // 전체 이름
  description: string;     // 기술 설명
  principle: string[];     // 작동 원리
  manufacturing: string[]; // 제조 공정
  advantages: string[];    // 장점
  disadvantages: string[]; // 단점
  applications: string[];  // 활용 분야
  images?: string[];       // 이미지 경로
}
```

이렇게 TypeScript 인터페이스로 정의하여, 모든 디스플레이 기술 데이터가 **일관된 구조**를 갖도록 했습니다.

### 3. 동적 라우팅 구현

Next.js의 동적 라우팅을 활용하여 `/displays/[id]` 경로로 각 기술의 상세 페이지를 구현했습니다.

예를 들어:
- `/displays/oled` → OLED 상세 페이지
- `/displays/microled` → MicroLED 상세 페이지

이를 통해 **6개의 컴포넌트를 만들 필요 없이**, 하나의 템플릿으로 모든 페이지를 렌더링할 수 있었습니다.

### 4. 반응형 디자인

Tailwind CSS의 반응형 유틸리티를 사용하여:
- **모바일**: 세로 스크롤 중심의 단일 컬럼 레이아웃
- **태블릿**: 2컬럼 그리드
- **데스크톱**: 3컬럼 그리드와 사이드바

모든 디바이스에서 최적화된 사용자 경험을 제공합니다.

## 💡 주요 구현 기술 포인트 (1분)

### 1. 성능 최적화

**이미지 최적화**
- Next.js의 Image 컴포넌트를 사용하여 자동으로 WebP 변환 및 lazy loading 구현
- 각 디스플레이 기술마다 여러 이미지가 있지만, 필요할 때만 로드됩니다

**코드 스플리팅**
- 각 디스플레이 상세 페이지는 독립적으로 분리되어, 초기 번들 크기를 최소화했습니다

**Turbopack**
- Next.js 15의 Turbopack을 사용하여 개발 서버 시작 시간을 크게 단축했습니다

### 2. SEO 최적화

```typescript
export const SEO_CONFIG = {
  defaultTitle: '충북보건과학대학교 COSS혁신융합대학사업단 | 차세대 디스플레이 기술',
  description: '차세대 디스플레이 기술 학습 웹사이트...',
  keywords: ['충북보건과학대학교', 'COSS', 'OLED', 'QLED', 'MicroLED', ...]
}
```

각 페이지마다 적절한 메타 태그를 설정하여 검색 엔진 최적화를 구현했습니다.

### 3. 접근성 (Accessibility)

- **시맨틱 HTML**: `<article>`, `<section>`, `<nav>` 등 의미있는 태그 사용
- **ARIA 레이블**: 스크린 리더를 위한 적절한 레이블 추가
- **키보드 네비게이션**: Tab, Enter 키로 모든 기능 접근 가능
- **대비율**: WCAG 2.1 AA 기준 충족하는 색상 사용

### 4. 에러 처리

React의 Error Boundary를 구현하여, 런타임 에러가 발생해도 전체 앱이 중단되지 않고 적절한 에러 메시지를 표시합니다.

```typescript
<ClientErrorBoundary>
  {children}
</ClientErrorBoundary>
```

## 📌 예상 질문 & 답변

### Q1: 왜 Next.js를 선택했나요?

**A**: Next.js는 React 기반의 풀스택 프레임워크로, 다음과 같은 이유로 선택했습니다:
1. **SSR/SSG 지원**: SEO 최적화가 중요한 교육용 웹사이트에 적합
2. **App Router**: 직관적인 파일 기반 라우팅으로 개발 생산성 향상
3. **Image 최적화**: 자동 이미지 최적화로 성능 개선
4. **Vercel 배포**: 원클릭 배포 및 CI/CD 자동화

### Q2: 데이터베이스는 어떻게 사용하나요?

**A**: 현재는 정적 데이터(`displayData.ts`)를 사용하고 있지만, Neon PostgreSQL을 연결해두었습니다. 향후 계획으로:
- 사용자 학습 진도 저장
- 퀴즈 결과 저장
- 댓글/질문 게시판
- 관리자 CMS 기능

을 추가할 예정입니다.

### Q3: 모바일 최적화는 어떻게 했나요?

**A**:
1. **반응형 디자인**: Tailwind CSS의 `sm:`, `md:`, `lg:` 브레이크포인트 사용
2. **터치 최적화**: 버튼 크기 44x44px 이상 확보
3. **모바일 메뉴**: 햄버거 메뉴와 슬라이드 네비게이션
4. **성능**: Lighthouse 모바일 점수 90+ 달성
5. **이미지**: WebP 포맷과 lazy loading으로 데이터 사용량 최소화

### Q4: 접근성을 어떻게 보장했나요?

**A**:
1. **시맨틱 HTML**: 스크린 리더가 이해할 수 있는 구조
2. **키보드 네비게이션**: Tab 키로 모든 요소 접근 가능
3. **ARIA 레이블**: `aria-label`, `aria-describedby` 등 적절히 사용
4. **색상 대비**: WCAG 2.1 AA 기준 충족 (4.5:1 이상)
5. **Focus 표시**: 키보드 사용자를 위한 명확한 포커스 표시

### Q5: 성능 최적화는 어떻게 했나요?

**A**:
1. **코드 스플리팅**: 동적 라우팅으로 각 페이지 독립적으로 로드
2. **이미지 최적화**: Next.js Image 컴포넌트로 자동 최적화
3. **폰트 최적화**: Pretendard 폰트를 로컬에서 로드하여 FOUT 방지
4. **Turbopack**: 빠른 번들링과 개발 서버
5. **Lazy Loading**: Framer Motion의 `viewport` 옵션으로 뷰포트 내 요소만 애니메이션
